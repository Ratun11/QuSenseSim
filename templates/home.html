What QuSenseSim Provides

QuSenseSim is designed to help users study how quantum sensing systems behave under realistic noisy conditions, while making it easy to inspect mitigation strategies, sensor heterogeneity, and collaboration behavior.

Interactive Dashboard

Create and place sensors in a visual workspace, adjust environment parameters, assign circuits, and observe changes in sensing behavior in real time.

Noise-Aware Simulation

Explore gate noise, measurement noise, device noise, environment noise, and source effects under a Markovian approximation.

Mitigation Techniques

Compare execution-time mitigation techniques and post-processing mitigation techniques, both uniformly across sensors and in personalized sensor-specific configurations.

Collaboration Modes

Evaluate Independent, Entangled-Style, and Federated-Style sensing setups and inspect how collaboration changes performance under noise.

Time-Based Analysis

Study metric evolution over time using sensing duration and T1/T2-style decay behavior, with graphs and tables for easier interpretation.

Research-Oriented Output

Generate clean visual comparisons, sensor-level summaries, and export-friendly results that are suitable for reports, demos, and research presentation.

How to Use the Platform

The workflow is simple and is designed to guide the user from interactive setup to detailed analysis.

Step 1 — Configure the System

  1. Open the Dashboard.
  2. Add sensors and place them in the workspace.
  3. Adjust field strength, phase shift, and global environment noise.
  4. Optionally add source devices such as cooling, heating, shielding, or interference.

Step 2 — Assign Sensing Structure

  1. Select or build a circuit.
  2. Assign circuits to sensors.
  3. Control sensor-specific noise values where needed.
  4. Run the simulation to update workspace metrics.

Step 3 — Choose Mitigation

  1. Go to the Analysis page.
  2. Select collaboration mode(s).
  3. Choose uniform mitigation or personalized sensor mitigation.
  4. Apply execution-time and/or post-processing mitigation techniques.

Step 4 — Inspect the Results

  1. Compare FI, CRB, estimate, and effective noise.
  2. Inspect graph and table pairs.
  3. Review sensor-level performance.
  4. Use exports for reports and presentation workflows.

What to Expect

The platform is intended to make complex sensing behavior easier to visualize and compare.

  • Visual sensor placement and source-aware workspace interaction
  • Real-time updates of sensing metrics using backend-connected simulation
  • Noise-aware comparison across multiple mitigation strategies
  • Uniform and personalized mitigation workflows
  • Graph + table benchmarking for cleaner interpretation
  • Sensor-level detail views for heterogeneous systems
  • Export-ready analysis output for research and demo use
